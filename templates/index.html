🛡️ Đề tài CF10 - Lỗ Hổng File Upload

Môn: FIT4012 - Cyber Fortress

🚨 Hệ thống Lỗi (Vulnerable)

Mô phỏng máy chủ không kiểm tra kích thước, định dạng, nội dung hay tên file. Rất dễ bị Hacker khai thác up Shell.

🛡️ Hệ thống Vá Lỗi (Secure)

Phòng thủ đa lớp: Giới hạn 2MB, Whitelist đuôi ảnh, kiểm tra chữ ký Byte (MIME) và sinh mã tên file ngẫu nhiên.

📌 Danh sách Test Case Yêu Cầu Demo

Mã Test Kịch bản kiểm thử (Input) Bên Hệ thống Lỗi (Vuln) Bên Hệ thống Vá (Secure)
TC-01: Ảnh hợp lệ Upload file ảnh .jpg bình thường, dung lượng nhẹ. Nhận (Lưu tên gốc) Nhận (Đổi tên ngẫu nhiên UUID)
TC-02: Giả mạo đuôi Đổi file mã độc shell.php thành anh.jpg để lừa hệ thống. Nhận file Shell Chặn (Phát hiện Header sai)
TC-03: Tấn công DoS Upload file rác dung lượng khổng lồ (> 2MB). Nhận (Gây đầy ổ đĩa) Chặn (Vượt dung lượng)
TC-04: Sai định dạng Ngang nhiên upload file không phải ảnh (VD: trojan.exe, .txt). Nhận (Chứa chấp mã độc) Chặn (Bởi lớp Whitelist)
TC-05: Đa đuôi file Up file shell.php.jpg để lừa bộ lọc Whitelist kém. Nhận file Shell Chặn (Bởi lớp đọc Header ảnh)
TC-06: File vô danh Up file không có đuôi mở rộng (không có dấu chấm). Nhận file rác Chặn (Sai định dạng chuẩn)
